6. **Inline Page Builder** — In-page editing for public hotel pages
7. **Room Management** — Room list, room details, guest assignment, QR codes
8. **Room Bookings** — Multi-step guest booking flow, availability checking, Stripe payment, booking holds with countdown
9. **Staff Booking Management** — Booking list with advanced filters, approve/decline, precheckin links, overstay management

12. **Breakfast Service** — Separate breakfast ordering flow and staff management
13. **Restaurant Management** — Restaurant CRUD, floor plan blueprints, dining table management

15. **Maintenance** — Request submission form, request list with status tracking
16. **Hotel Info / Good-to-Know** — Category-based info CRUD, QR code generation for each category
17. **Guest Management** — Guest list, guest edit, guest assignment to rooms
18. **Staff Management** — Staff list, create, details, profile, permissions management
19. **Staff Attendance** — Clock in/out, department view, roster analytics, period management

23. **Staff Social Feed** — Posts with comments, post composer

27. **Stock Tracker** — Dashboard, items, stocktakes, movements, periods, profitability, cocktails

44. **Booking Overstay Management** — Overstay alerts, acknowledge, extend stay (with idempotency)

## 6. ALL AUTH FLOWS

### 6.1 Staff Login Flow
```
Login.jsx → useLogin() hook → POST /staff/login/
  → Response: { token, user, hotel_slug, access_level }
  → AuthContext stores user
  → authStore.setAuthUser(user, token) [non-React bridge]
  → localStorage: user, viewMode, selectedHotel
  → Redirect to /staff/{hotel_slug}/feed
```

### 6.2 Staff Registration Flow
```
QR Code scanned → /register?token=X&hotel=Y
  → RegisterWithToken validates query params
  → Register.jsx → POST /staff/register/
  → Redirect to /registration-success
  → Explains: manager must approve before login works
```

### 6.3 Password Reset Flow
```
ForgotPassword.jsx → POST /staff/password-reset/ (email)
  → Email received with link → /reset-password/:uid/:token/
  → ResetPassword.jsx → POST /staff/password-reset-confirm/
  → Redirect to /login
```

### 6.4 Guest Token Flow
```
Booking Confirmation email → Link with ?token=X&email=Y
  → guestToken.js: resolveGuestToken (URL > localStorage)
  → guestBookingTokens.js: resolveGuestBookingToken (payload > localStorage > URL)
  → Token used in X-Guest-Chat-Session header or ?token= query
  → Session scoped to hotel + booking
```

### 6.5 Player Token Flow (Games)
```
First game play → playerToken.js: getPlayerToken()
  → Generate: player_{timestamp}_{random}
  → Store in localStorage
  → Used for tournament identity without login
```

### 6.6 Route Protection
```
ProtectedRoute.jsx:
  Layer 1: Is user authenticated? → No → /login
  Layer 2: (if ENABLE_ROUTE_PERMISSIONS=true)
    staffAccessPolicy.canAccessStaffPath({ pathname, user, requiredSlug })
      → Superuser: bypass all
      → Admin routes: require super_staff_admin
      → Nav-slug routes: user.allowed_navs must include requiredSlug
      → Deny-by-default for unmapped routes
```

### 6.7 Axios Auth Interceptors
```
api instance → Request interceptor:
  authStore.getAuthUser() → Authorization: Token {token}
  selectedHotel → X-Hotel-ID, X-Hotel-Slug

Response interceptor:
  401 → clear auth, redirect to /login
  Network error → redirect to /no-internet
```

## 14. HIGH-RISK INCONSISTENCIES

### CRITICAL

1. **3 Hotel Param Names** — `:hotelSlug`, `:hotel_slug`, `:hotelIdentifier` used interchangeably across routes. A refactor that misses one style will break routes.

2. **Guest Token Dual System** — `guestToken.js` (global) and `guestBookingTokens.js` (booking-scoped) manage guest auth in parallel. If a component uses the wrong one, auth fails silently.

3. **Mark-Read Endpoint Mismatch** — Staff chat uses `/mark-read/` (hyphen), guest chat uses `/mark_read/` (underscore). A copy-paste error would cause silent failures.

4. **Page Builder Duplication** — Two separate APIs (`sectionEditorApi.js` and `staffApi.js`) both manage public hotel page sections/elements with overlapping CRUD operations. Editing through one may not reflect in the other.

5. **Chat State Split** — chatStore is supposedly the "single source of truth" but ChatContext, StaffChatContext, and MessengerContext all wrap it differently. A new developer could easily create a fourth consumer that conflicts.

### HIGH

6. **Order Count Competition** — `useOrderCount` (API polling) and `useOrderCountWebSocket` (WebSocket) are parallel implementations. If both run simultaneously, they could show different counts.

7. **Booking Data Dual-Source** — React Query cache and roomBookingStore both hold booking data. The bridging is done via debounced `invalidateQueries` (500ms delay) — during that window, the two sources disagree.

8. **Console.log Pollution** — 200+ console.log statements across production code. Several are in critical paths (Firebase setup, chat handling, authentication). These leak PII and tokens to browser console.

9. **Voice API Disabled But UI Active** — Voice recording UI components are fully functional but API calls return mock failures. A user can record audio that goes nowhere.

10. **Debug Panels in Production** — 7 debug panels and 15+ window debug globals may be accessible in production builds. The RealtimeDebugPanel and ChatRealtimeDebugPanel don't appear to have ENV-gated rendering.

### MEDIUM

11. **PUT vs PATCH Inconsistency** — `shiftLocations.js` uses PUT while all other update operations use PATCH. This implies different backend semantics (full replacement vs partial update) that may not be intentional.

12. **Empty Directory Stubs** — `pages/hotel-operations/bookings/` and `rooms/` are empty directories suggesting planned features that were never built or were moved elsewhere.

13. **Deprecated Navbar Still in Codebase** — `Navbar.jsx` is marked "Legacy" but still importable. If someone `Ctrl+P` searches for "Navbar", they'll find the wrong one first.

14. **GallerySectionView.jsx.backup** — A `.backup` file exists in `components/sections/`. This is not gitignored and indicates manual versioning instead of proper VCS usage.

15. **guestNotifications.js + .jsx Duplicate** — Identical code in two files with different extensions. Imports could resolve to either depending on bundler config.
